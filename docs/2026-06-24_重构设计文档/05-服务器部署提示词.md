## 部署提示词正文

````markdown
# 任务：部署 Simprint 自托管服务器

你是 root 用户，在一台全新的 Ubuntu 24.04 服务器（2核2G/40G）上部署 Simprint 桌面应用的自托管后端服务。请严格按以下步骤执行，每步执行后报告结果，失败立即停下排查。

## 服务器信息
- IP: 216.36.124.98
- 系统: Ubuntu 24.04.1 x64
- 域名: <YOUR_DOMAIN>（已解析到本机 IP）
- 邮箱: <YOUR_EMAIL>

## Phase 0：安全加固（必须最先执行）

1. 修改 root 密码为强密码（原密码已暴露，必须改）：
   ```bash
   passwd root
   ```

2. 创建部署专用非 root 用户 `simprint`，加入 sudo 组：
   ```bash
   adduser simprint
   usermod -aG sudo simprint
   ```

3. 配置 SSH key 登录（若你已有公钥），并禁用密码登录：
   ```bash
   mkdir -p /home/simprint/.ssh
   # 把调用方提供的公钥写入 /home/simprint/.ssh/authorized_keys
   chmod 700 /home/simprint/.ssh
   chmod 600 /home/simprint/.ssh/authorized_keys
   chown -R simprint:simprint /home/simprint/.ssh
   # 编辑 /etc/ssh/sshd_config 设 PasswordAuthentication no
   systemctl restart sshd
   ```

4. 配置 UFW 防火墙，仅放行必要端口：
   ```bash
   ufw default deny incoming
   ufw default allow outgoing
   ufw allow 22/tcp
   ufw allow 80/tcp
   ufw allow 443/tcp
   ufw --force enable
   ```

5. 安装 fail2ban 防爆破：
   ```bash
   apt update && apt install -y fail2ban
   systemctl enable --now fail2ban
   ```

6. 启用自动安全更新：
   ```bash
   apt install -y unattended-upgrades
   dpkg-reconfigure -plow unattended-upgrades
   ```

## Phase 2：安装 PostgreSQL

```bash
apt install -y postgresql postgresql-contrib
systemctl enable --now postgresql

# 创建数据库与用户
sudo -u postgres psql <<EOF
CREATE USER simprint_user WITH PASSWORD '<YOUR_DB_PASSWORD>';
CREATE DATABASE simprint_db OWNER simprint_user;
GRANT ALL PRIVILEGES ON DATABASE simprint_db TO simprint_user;
\c simprint_db
CREATE EXTENSION IF NOT EXISTS "pgcrypto";  -- for gen_random_uuid()
EOF
```

按 03 文档的 Schema 建表（users / sessions / referral_codes / environments / proxies / groups / tags / workspaces）。完整建表 SQL 见 03 文档第四节，逐条执行。

## Phase 3：安装 Go 与部署服务端

```bash
# 安装 Go 1.22
cd /tmp
wget https://go.dev/dl/go1.22.0.linux-amd64.tar.gz
tar -C /usr/local -xzf go1.22.0.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> /etc/profile
source /etc/profile
go version
```

服务端代码需按 03 文档实现（端点规格 + 加密协议）。核心实现要点：

1. **RSA 密钥对生成与持久化**（服务端长期密钥对，非每次生成）：
   ```go
   // 生成 RSA-2048 密钥对，存到 /etc/simprint/keys/
   // 公钥通过 GET /secret/public/key 返回（PKCS1 PEM 格式）
   ```

2. **加密协议实现**（严格对齐 03 文档第二节）：
   - 请求解密：RSA 私钥解密 AES 密钥 → AES-GCM 解密 body → 校验 api_secret
   - 响应加密：客户端公钥 RSA 加密新 AES 密钥 → AES-GCM 加密响应

3. **必须实现的端点**：
   - `GET /secret/public/key` → 返回 RSA 公钥（不加密）
   - `POST /api/v1/users/login` → 登录（支持 Basic + RememberPassword）
   - `POST /api/v1/users/register` → 注册（含邀请码）
   - `POST /api/v1/users/refresh-credentials` → 刷新 token
   - `POST /api/v1/local-api/*` → 业务资源透传
   - `POST /update/api/v1/versions/check` → 更新检查
   - `GET /update/latest.json` → 主程序更新清单
   - `GET /update/simprint-runtime/latest.json` → runtime 更新清单

4. **配置**（/etc/simprint/config.env）：
   ```env
   DB_HOST=127.0.0.1
   DB_PORT=5432
   DB_USER=simprint_user
   DB_PASSWORD=<YOUR_DB_PASSWORD>
   DB_NAME=simprint_db
   API_SECRET=<YOUR_API_SECRET>
   SERVER_PORT=8080
   RSA_PRIVATE_KEY_PATH=/etc/simprint/keys/server_private.pem
   RSA_PUBLIC_KEY_PATH=/etc/simprint/keys/server_public.pem
   ```

5. **systemd 服务**（/etc/systemd/system/simprint.service）：
   ```ini
   [Unit]
   Description=Simprint Self-hosted Server
   After=network.target postgresql.service

   [Service]
   Type=simple
   User=simprint
   WorkingDirectory=/opt/simprint
   EnvironmentFile=/etc/simprint/config.env
   ExecStart=/opt/simprint/bin/simprint-server
   Restart=always
   RestartSec=5

   [Install]
   WantedBy=multi-user.target
   ```

   ```bash
   systemctl daemon-reload
   systemctl enable --now simprint
   systemctl status simprint
   ```

## Phase 4：Nginx 反代 + TLS

```bash
# Nginx 配置 /etc/nginx/sites-available/simprint
```

配置内容：
```nginx
server {
    listen 80;
    server_name <YOUR_DOMAIN>;
    # 用于 Let's Encrypt 验证
    location /.well-known/acme-challenge/ {
        root /var/www/html;
    }
    location / {
        return 301 https://$host$request_uri;
    }
}

server {
    listen 443 ssl http2;
    server_name <YOUR_DOMAIN>;

    ssl_certificate /etc/letsencrypt/live/<YOUR_DOMAIN>/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/<YOUR_DOMAIN>/privkey.pem;

    # 客户端主 API
    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 更新服务
    location /update/ {
        proxy_pass http://127.0.0.1:8080/update/;
        proxy_set_header Host $host;
    }

    # 限流（防滥用）
    limit_req zone=api burst=20 nodelay;
}

# 在 http 块定义限流区
# limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;
```

## Phase 5：验证

1. 公钥端点可达：
   ```bash
   curl https://<YOUR_DOMAIN>/api/v1/secret/public/key
   # 应返回 {public_key: "-----BEGIN RSA PUBLIC KEY-----..."}
   ```

2. 服务进程在跑：
   ```bash
   systemctl status simprint
   curl http://127.0.0.1:8080/health
   ```

3. 数据库连接正常：
   ```bash
   sudo -u postgres psql -d simprint_db -c "SELECT count(*) FROM users;"
   ```

4. TLS 证书有效：
   ```bash
   curl -vI https://<YOUR_DOMAIN>/api/v1/secret/public/key 2>&1 | grep -E "SSL|expire"
   ```

## Phase 6：生成客户端配置

部署完成后，生成客户端 `config.production.toml`（供客户端使用）：
```toml
[server]
base_url = "https://<YOUR_DOMAIN>/api/"
version = "v1"
secret_key = "<YOUR_API_SECRET>"

[updater]
check_url = "https://<YOUR_DOMAIN>/update/api/v1/versions/check"
latest_json_url = "https://<YOUR_DOMAIN>/update/latest.json"
runtime_latest_json_url = "https://<YOUR_DOMAIN>/update/simprint-runtime/latest.json"
updater_temp_dir = "updates"

[webview]
downlaod_url = "https://<YOUR_DOMAIN>/update/webview-fixed.zip"
```

将此配置写入客户端 `src-tauri/config.production.toml`，重跑 build.rs 生成加密 bin，重新编译客户端。

## 执行要求
- 每个 Phase 执行后报告 `Status(DONE|DONE_WITH_CONCERNS|BLOCKED)` + 关键输出
- 任何命令失败立即停下，报告错误，不自行跳过
- 密码/密钥不写入任何日志文件
- 全程用 root 或 simprint（sudo）执行
- 2核2G 资源有限，监控内存（free -h），若 PostgreSQL 内存占用过高调整 shared_buffers
````

## 使用说明

1. 将上方"部署提示词正文"整段复制
2. 替换所有 `<占位符>`（域名/数据库密码/API密钥/邮箱）
3. 粘贴到 ZCode 的 root 对话窗口执行
4. ZCode 会按 Phase 0→6 逐步执行并报告

## 资源约束提示

2核2G 服务器资源有限，需注意：
- PostgreSQL 默认配置可能吃 400MB+，必要时 `shared_buffers=128MB`
- Go 服务端空闲约 50-100MB，运行时按连接数增长
- 系统总内存控制在 1.5GB 以内，留缓冲
- 若后续业务量大（多用户/多环境数据），考虑加数据盘或升配
