## 一、布局架构现状

基于 `@slotkitjs/core` 插件系统，`src/App.tsx` 按 `mode`（main/splashscreen/syncer）渲染，`pluginLoader.loadAllPlugins()` 动态加载。

### 1.1 主窗口布局（`plugins/layouts/app-layout/`）
```
app-layout/src/
├── components/
│   ├── app-background.tsx          # 背景层
│   ├── app-sidebar.tsx             # 侧边导航（官方 +1105 行，v0.2.26 大改）
│   └── titlebar/                   # 标题栏（自定义窗口控件）
│       ├── titlebar.tsx            # 标题栏主组件
│       ├── user-menu.tsx           # 用户菜单
│       ├── workspace-selector.tsx  # 工作区切换
│       ├── notification-menu.tsx   # 通知
│       ├── download-menu.tsx       # 下载管理
│       ├── prepared-update-button.tsx  # 更新就绪按钮
│       ├── default-brand-slot.tsx      # 品牌位
│       ├── default-navigation-slot.tsx # 导航位
│       ├── default-status-slot.tsx     # 状态位
│       └── default-window-controls-slot.tsx  # 窗口控制（最小化/关闭）
├── api/（workspaces/teams/messages/workspace-quotas）
└── stores/（download-store/messages-store）
```

### 1.2 页面组织（`plugins/pages/`，20 个 slotkit 插件）
account-center / api-ai / audit / billing-center / browser-extensions / create-window / environment-manager / group-management / login / plan-selection / proxy-center / referral-program / register / reset-password / rpa-workflow / splashscreen / syncer / system-settings / team

各页通过 `extensionRegistry.contribute('routes', {...})` 注册路由。

### 3.2 系统托盘（`tray.rs`）
| 操作 | 当前实现 | 符合习惯 | 改进建议 |
|------|---------|---------|---------|
| 左键单击 | 显示/聚焦主窗口（未就绪显 splashscreen） | ✅ | 符合 Windows 习惯 |
| 右键菜单 | 显示/退出两项 | ⚠️ | 建议增加"设置""检查更新"项 |
| 双击 | 无处理 | ⚠️ | Windows 习惯双击显窗，当前左键单击已覆盖，可接受 |
| 气泡通知 | 无 | ⚠️ | 更新就绪建议托盘气泡提示 |

**改进**：托盘右键菜单增加常用项（显示/设置/检查更新/关于/退出），减少操作步数。

### 3.3 快捷键（缺失项）
当前**未注册全局/应用快捷键**（Grep 未命中 hotkey/shortcut/GlobalShortcut）。建议补充高频快捷键：

| 快捷键 | 功能 | 优先级 |
|--------|------|--------|
| `Ctrl+N` | 新建浏览器环境 | 高 |
| `Ctrl+,` | 打开设置 | 高 |
| `Ctrl+P` | 打开代理中心 | 中 |
| `Ctrl+Shift+M` | 启动/停止 MCP | 中 |
| `Ctrl+Q` | 退出 | 低 |

实现：`app/setup.rs` 注册 `tauri::plugin::GlobalShortcut` 或前端 `useKeyPress`。

### 5.2 首启流程（splashscreen → main）
`app/splashscreen.rs::init_startup` 流程：
```
前端就绪 → init(10%) → config(30%) → security(50%) → server(70%)
→ update-check(92%) → create_main_window → ready(100%)
```

| 复核项 | 检查内容 | 通过标准 |
|--------|---------|---------|
| 进度反馈 | 各阶段有进度事件 | splashscreen 显示百分比 |
| 失败处理 | server 失败 emit connection_failed | 不卡死，可重试 |
| 更新阻塞 | 有更新时启动安装并退出不创建主窗口 | 流程正确 |
| 自建服务器 | server(70%) 指向自建服务器拉公钥 | 成功 |

**改进**：server 阶段失败时，提供"切换服务器/检查网络"的明确指引，而非仅 connection_failed。

## 六、交互统一性深度复核

### 6.1 对话框/弹窗
- 统一用 `extended-dialog` / `formatted-dialog`（`src/components/`）
- 新增页面（browser-extensions/mihomo）必须复用，禁止自造弹窗

### 6.2 表格
- 统一用 `data-table`（`src/components/data-table/`）
- 分页/排序/筛选交互一致

### 6.3 图标
- 统一用 `lucide-react`（已依赖）
- 禁止混用其他图标库

### 6.4 主题
- `theme-provider` 统一明暗主题
- 新增页面适配主题切换
