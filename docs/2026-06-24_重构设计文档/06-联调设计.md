## 二、加密协议自测用例（服务端独立验证）

服务端部署后，先用以下用例独立验证加密协议实现正确，再联客户端。

### 用例 1：公钥端点
```bash

curl -s https://<YOUR_DOMAIN>/api/v1/secret/public/key | jq .
# 校验：public_key 字段以 "-----BEGIN RSA PUBLIC KEY-----" 开头
```

### 用例 2：加密往返（用 Go 测试脚本）
服务端写一个测试脚本，模拟客户端加密 → 服务端解密 → 服务端加密响应 → 模拟客户端解密：

```go
// test/crypto_roundtrip_test.go
func TestEncryptDecryptRoundtrip(t *testing.T) {
    // 1. 生成临时客户端 RSA 密钥对（模拟客户端启动）
    clientPriv, clientPub := generateRSAKeyPair()

    // 2. 获取服务端公钥
    serverPub := getServerPublicKey()

    // 3. 模拟客户端加密请求
    payload := map[string]interface{}{
        "email":     "test@example.com",
        "password":  "test123",
        "api_secret": "<YOUR_API_SECRET>",
        "public_secret_key": pemEncode(clientPub),
    }
    encrypted := clientEncrypt(payload, serverPub)

    // 4. 服务端解密
    decrypted, err := serverDecrypt(encrypted)
    assert.NoError(t, err)
    assert.Equal(t, "test@example.com", decrypted["email"])
    assert.Equal(t, "<YOUR_API_SECRET>", decrypted["api_secret"])

    // 5. 服务端加密响应（用客户端公钥）
    response := map[string]interface{}{"access_token": "xxx", "is_first_login": false}
    encryptedResp := serverEncrypt(response, clientPubPEM)

    // 6. 模拟客户端解密响应
    decryptedResp, err := clientDecrypt(encryptedResp, clientPriv)
    assert.NoError(t, err)
    assert.Equal(t, "xxx", decryptedResp["access_token"])
}
```

## 三、客户端 ↔ 服务器联调清单

### 阶段 A：登录链路

| 步骤 | 操作 | 预期 | 验证方法 |
|------|------|------|---------|
| A1 | 客户端 config.development.toml 改为自建服务器 | 配置加载成功 | `cargo tauri dev` 启动无配置错误 |
| A2 | 重跑 build.rs | config_encrypted.bin 重新生成 | 编译通过 |
| A3 | 客户端启动 | 拉取服务端公钥成功 | 启动日志无 `fetch_server_public_key` 失败 |
| A4 | 注册新账号 | 注册成功，返回 token | 数据库 users 表有记录 |
| A5 | 账密登录 | 登录成功，token 持久化 | `is_logged_in` 返回 true |
| A6 | 退出再"记住密码"登录 | refresh_token 自动登录 | keyring 有记录，重启自动登录 |
| A7 | token 过期自动刷新 | refresh_credentials 成功 | 长时间运行后业务请求不报 401 |

### 阶段 B：业务数据链路

## 四、故障排查矩阵

| 症状 | 可能原因 | 排查方法 |
|------|---------|---------|
| 客户端启动卡在"连接服务器" | 公钥拉取失败 | `curl https://<YOUR_DOMAIN>/api/v1/secret/public/key` |
| 登录 401 | api_secret 不符 | 核对客户端 config 的 secret_key 与服务端 API_SECRET |
| 登录解密失败 | RSA padding 不一致 | 读 `rsa.rs` 确认 padding，服务端对齐 |
| 业务请求 401 | access_token 过期且 refresh 失败 | 查服务端 refresh-credentials 日志 |
| 更新检查无响应 | check_url 不通 | curl 测试 `/update/api/v1/versions/check` |
| MCP 工具调用失败 | Local API 未起或鉴权失败 | 查 `mcp/manager.rs::fetch_local_api_runtime_config` |
